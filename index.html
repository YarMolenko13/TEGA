  • Extract, Transform, Load (Извлечение, Преобразование, Загрузка) – один из основных процессов в управлении хранилищами данных, который включает в себя:
  • • Извлечение данных из внешних источников
  • • Трансформация и очистка данных для обеспечения их соответствия потребностям бизнес-модели
  • • Загрузка данных в хранилище
  • С точки зрения процесса ETL, архитектуру хранилища данных можно представить в виде трёх компонентов:
  • • Источник данных: содержит структурированные данные в виде таблиц, совокупности таблиц или просто файл
  • • Промежуточная область: содержит вспомогательные таблицы, создаваемые временно, исключительно для организации процесса выгрузки
  • • Получатель данных: хранилище данных или база данных, в которую должны быть помещены извлечённые данные
  • • ETL–процедура является не только процессом переноса данных из одного приложения в другое, но и инструментом подготовки данных к анализу
    Витрины могут строиться как непосредственно над учетными системами (системами–источниками), так и над хранилищем данных банка. Аккумулируют информацию, необходимую для анализа, организуя ее в виде интегрированного, согласованного и зависимого от времени набора данных. Витрины агрегированных данных представляют собой предрассчитанные на основе детализированных данных агрегаты, многомерные денормализованные представления с необходимой степенью агрегации по аналитическим показателям, с учетом их взаимодействия между собой (кросс–связи, иерархии, агрегирование) и используются для решения задач быстрой аналитической обработки данных
  • В рамках предметных областей может быть реализовано несколько видов банковской отчетности, за счет использования множества иерархических структур и различных аналитических признаков–классификаторов:

  • • Банковская Отчетность:
  • 1. Управленческая
  • 2. МСФО
  • 3. Обязательная (в том числе расчет нормативов)

  • • Аналитика:
  • 1. Обеспечение на витринах данных возможностей
  • 2. Формирования на основе единой логической модели данных произвольных аналитических выборок на произвольную дату
  • 3. Построения таблиц среза, изменения уровней агрегирования данных «на лету»
  • 4. Детализации данных любого уровня агрегирования в связанные по аналитическим разрезам наборы отчетных данных (сделки, проводки)
Проектный опыт

Сотрудники подразделения принимали активное участие в проектах развития и сопровождения банковской отчетности более 20 российских банков с 2000 г.

С 2011 г. наша команда является партнером в проектах развития аналитики и отчетности Альфа–Банка. За время сотрудничества нашими специалистами успешно реализовано множество проектов по повышению качества формирования управленческой отчетности и отчетности по МСФО: были доработаны витрины данных Розничного и Корпоративного Блоков, витрина Системы коррекций, витрина Казначейства, автоматизированы процессы расчета:

  • • Портфелей ценных бумаг, средств клиентов, прочих требований и обязательств
  • • Стоимости капитала (управленческого и финального) с учетом расчета RWA по стандартам Базель
  • • Различных корректировок для трансформации отчетности в МСФО (факторинг, консолидация, распределение остатков на сводных счетах, инкассация)
  • • Управленческой отчетности по различным аналитикам (Валютам, Бизнес-Блокам, Дивизионам, Городам и т.д.): Баланс, P&L, Средние портфели, Средние %%, KPI
  • • Аналитической отчетности для Казначейства.

В 2019 г. началось и в настоящее время успешно продолжается сотрудничество компании и ПАО Росбанк в обрасти консалтинга, бизнес- и системного анализа в проекте разработки Платформы он-лайн обработки данных (ODPP – система сбора, хранения, обработки и интеграции данных различных учетных и аналитических систем–источников на базе Hadoop).

С 2020 г. компания обеспечивает техническое сопровождение Аналитической системы на платформе Oracle BI АО ВТБ-Лизинг. В рамках сопровождения реализован процесс расчета аллокаций расходов, автоматизирован расчет лимитов, лизингового портфеля, справедливой ставки доходности сделки.
